# ควิซออนไลน์ + ระบบตรวจจับการออกจากหน้าจอ

เว็บทำแบบทดสอบ (1 โจทย์ 4 ช้อย) คล้าย Google Form แต่เพิ่มฟีเจอร์พิเศษ:
**ผู้สร้างดูได้ว่าผู้ตอบคนไหนปัด/สลับ/ย่อออกจากหน้าจอระหว่างทำ กี่ครั้ง เมื่อไหร่ และหายไปนานเท่าไร**

## วิธีรัน (ไม่ต้องติดตั้งอะไรเพิ่ม)

```bash
cd quiz-app
node server.js
```

แล้วเปิดเบราว์เซอร์ไปที่ **http://localhost:3000**

> ต้องมี Node.js ติดตั้งอยู่ (เครื่องนี้มี v24 แล้ว) — ไม่ใช้ npm install ใด ๆ

## การใช้งาน

1. **สร้างควิซ** ที่หน้าแรก → ใส่คำถาม 4 ช้อย เลือกคำตอบที่ถูก
2. กด "สร้างควิซ" จะได้ **2 ลิงก์**:
   - 🔗 *ลิงก์ตอบ* — ส่งให้คนอื่นเข้าทำ (`/take?id=...`)
   - 📊 *ลิงก์ดูผล* — เก็บไว้ส่วนตัว ดูคะแนน + พฤติกรรมออกจากจอ (`/admin?id=...&token=...`)
3. ผู้ตอบใส่ชื่อ → ทำควิซ → ส่งคำตอบ
4. เปิดลิงก์ดูผลเพื่อดูตาราง (อัปเดตอัตโนมัติทุก 10 วิ)

## ระบบตรวจจับการออกจากจอทำงานยังไง

ใช้ web API ฝั่งเบราว์เซอร์ 3 อย่างส่งสัญญาณกลับเซิร์ฟเวอร์:
- `visibilitychange` — สลับแท็บ / ย่อจอ / ปัดออกบนมือถือ
- `blur` — สลับไปหน้าต่าง/แอปอื่นบนเดสก์ท็อป
- `pagehide` + `navigator.sendBeacon` — ปิดแท็บ/รีเฟรช (ส่งได้แม้ตอนกำลังปิด)

แต่ละเหตุการณ์ผูกกับ `sessionId` ของผู้ตอบ ผู้สร้างจึงเห็นไทม์ไลน์รายคน

## โครงสร้างไฟล์

```
quiz-app/
├─ server.js          เซิร์ฟเวอร์ Node (ไม่มี dependency) + API
├─ data/db.json       ฐานข้อมูล (สร้างอัตโนมัติตอนรัน)
└─ public/
   ├─ index.html      หน้าสร้างควิซ
   ├─ take.html       หน้าทำควิซ (มีตัวตรวจจับการออกจากจอ)
   ├─ admin.html      แดชบอร์ดดูผล
   └─ style.css
```

## Deploy ขึ้นเซิร์ฟเวอร์กลาง (Render.com)

เพื่อให้หลายคนเข้าใช้งานพร้อมกันผ่านลิงก์เดียวได้ ทำตามนี้:

1. **สร้าง repo บน GitHub** แล้ว push โค้ดนี้ขึ้นไป:
   ```bash
   git remote add origin https://github.com/<username>/quiz-app.git
   git commit -m "quiz app"
   git branch -M main
   git push -u origin main
   ```
2. เข้า [render.com](https://render.com) → สมัคร/ล็อกอินด้วย GitHub → **New +** → **Blueprint**
3. เลือก repo `quiz-app` — Render จะอ่าน `render.yaml` ในโปรเจกต์นี้ให้อัตโนมัติ
   (ตั้งค่า start command, persistent disk สำหรับเก็บ `data/db.json` ไว้ให้แล้ว)
4. กด **Apply** รอ deploy เสร็จ (~1-2 นาที) จะได้ลิงก์แบบ `https://quiz-app-xxxx.onrender.com`
5. ใช้ลิงก์นั้นแทน `localhost:3000` ได้เลย — ทุกคนเข้าพร้อมกันได้

> ⚠️ **สำคัญ:** ฟีเจอร์ persistent disk ต้องใช้แพลน **Starter ขึ้นไป (มีค่าใช้จ่าย ~$7/เดือน)**
> ถ้าใช้แพลนฟรี ข้อมูลจะหายทุกครั้งที่ redeploy/restart เซิร์ฟเวอร์ (ไฟล์ระบบเป็นแบบชั่วคราว)
> ถ้าต้องการใช้ฟรีจริง ๆ ให้พิจารณาย้ายไปเก็บข้อมูลใน DB ภายนอกแทน (เช่น Supabase/Neon ฟรี tier)

**ทางเลือกอื่น:** Railway.app ใช้วิธีคล้ายกัน (เชื่อม GitHub → เพิ่ม Volume mount ที่ `/var/data` → ตั้ง env `DATA_DIR=/var/data`)

## ข้อจำกัด / หมายเหตุ

- เก็บข้อมูลในไฟล์ `data/db.json` เหมาะกับการใช้งานส่วนตัว/ห้องเรียน ไม่ใช่ระดับ production ขนาดใหญ่
- ระบบตรวจจับเป็น "ตัวช่วยดูพฤติกรรม" ไม่ใช่ระบบกันโกงกันน้ำ (ผู้ใช้ที่ตั้งใจเลี่ยงยังทำได้) แต่ครอบคลุมกรณีสลับแท็บ/ย่อจอทั่วไปได้ดี
- ลิงก์ดูผลใครมี token ก็เข้าดูได้ — อย่าเผยแพร่ลิงก์ admin
